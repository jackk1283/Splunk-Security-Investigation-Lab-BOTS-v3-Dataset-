# Splunk-Security-Investigation-Lab-BOTS-v3-Dataset-
This investigation analyzed suspicious outbound network traffic in the BOTS v3 dataset using Splunk Enterprise. The internal host 192.168.70.186 was identified as repeatedly initiating connections to the external IP address 45.77.53.176. Host network activity was reviewed by protocol to understand the types of traffic generated, and log analysis confirmed that active communication between the internal host and the external destination was occurring. The external IP was further investigated through geolocation and WHOIS analysis to identify the hosting provider and geographic location of the server. DNS activity was examined to determine whether the host attempted to resolve a domain associated with the destination IP. The communication protocol and destination port were validated to ensure the traffic matched expected behavior, and the volume of data transmitted between the host and the destination was analyzed to assess whether significant data transfer or potential exfiltration occurred. A timeline of events and connection frequency analysis revealed repeated communication over time, and further analysis of the intervals between connections showed a consistent and automated pattern. This regular timing pattern is characteristic of beaconing behavior commonly associated with command-and-control communication used by malware to maintain contact with a remote server.
