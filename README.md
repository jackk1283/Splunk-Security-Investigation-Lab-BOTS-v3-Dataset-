# Splunk-Security-Investigation-Lab-BOTS-v3-Dataset-
This investigation analyzed suspicious outbound network traffic in the BOTS v3 dataset using Splunk Enterprise. The internal host 192.168.70.186 was identified as repeatedly initiating connections to the external IP address 45.77.53.176. Host network activity was reviewed by protocol to understand the types of traffic generated, and log analysis confirmed that active communication between the internal host and the external destination was occurring. The external IP was further investigated through geolocation and WHOIS analysis to identify the hosting provider and geographic location of the server, but the WHOIS lookup returned no useful information within the dataset, which increased suspicion around the destination infrastructure. DNS activity was then examined to determine whether the host attempted to resolve a domain associated with the destination IP, but no DNS events were found for that IP, indicating the host connected directly to the address rather than resolving a legitimate service domain. Protocol analysis showed very little HTTP/HTTPS activity associated with the communication, which is unusual for normal web services, while destination port analysis revealed that most connections occurred over port 443 with a small number using port 3333, a non-standard port that can sometimes be associated with backdoor or command-and-control services. Data transfer between the host and the destination was analyzed to assess communication behavior, and timeline visualizations showed repeated connections occurring over time. Further analysis of the intervals between connections revealed a consistent and automated pattern, which is characteristic of beaconing behavior commonly associated with command-and-control communication used by malware to maintain contact with a remote server.
